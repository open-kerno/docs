TypeScript Implementation

All examples use date-fns for manipulation and Node.js built-ins for formatting.

npm install date-fns date-fns-tz

Parse and Format​

Parse any ISO 8601 string and return a UTC Date object.

import { parseISO, isValid, formatISO } from 'date-fns';

// Input:  "2024-03-10T14:30:00.000Z"
// Output: Date object (UTC)

export function parseUtcDate(isoString: string): Date {
  return parseISO(isoString);
}

// Input:  Date object
// Output: "2024-03-10T14:30:00.000Z"

export function toIsoUtcString(date: Date): string {
  return formatISO(date, { representation: 'complete' });
}

Validate ISO 8601 Input​

Typed function that validates an incoming date string before use.

import { parseISO, isValid } from 'date-fns';

interface DateValidationResult {
  valid: boolean;
  date: Date | null;
  error: string | null;
}

export function validateIsoDate(input: string): DateValidationResult {
  if (typeof input !== 'string' || input.trim() === '') {
    return { valid: false, date: null, error: 'Input must be a non-empty string.' };
  }

  const parsed = parseISO(input);

  if (!isValid(parsed)) {
    return { valid: false, date: null, error: `"${input}" is not a valid ISO 8601 date.` };
  }

  return { valid: true, date: parsed, error: null };
}

Usage:

const result = validateIsoDate('2024-03-10T14:30:00.000Z');
// { valid: true, date: Date, error: null }

const bad = validateIsoDate('March 10, 2024');
// { valid: false, date: null, error: '"March 10, 2024" is not a valid ISO 8601 date.' }

Normalize to UTC​

Convert a local-timezone datetime string (with offset) to UTC before storing.

import { parseISO, formatISO } from 'date-fns';
import { toZonedTime, fromZonedTime } from 'date-fns-tz';

// Input:  "2024-03-10T09:00:00+05:30" (user's local time with offset)
// Output: "2024-03-10T03:30:00.000Z"  (UTC)

export function normalizeToUtc(isoStringWithOffset: string): string {
  const parsed = parseISO(isoStringWithOffset);
  return parsed.toISOString(); // JavaScript Date is always UTC internally
}

// Input:  UTC Date object + IANA timezone string
// Output: formatted local-time string for display only (never store this)

export function formatForDisplay(utcDate: Date, userTimezone: string): string {
  const zonedDate = toZonedTime(utcDate, userTimezone);
  return formatISO(zonedDate, { representation: 'complete' });
}

Usage:

const utcString = normalizeToUtc('2024-03-10T09:00:00+05:30');
// "2024-03-10T03:30:00.000Z"

const display = formatForDisplay(new Date('2024-03-10T03:30:00.000Z'), 'Asia/Kolkata');
// "2024-03-10T09:00:00+05:30" — for UI only, never write back to DB

Express Middleware: Validate Date Body Fields​

Middleware that validates all ISO 8601 date fields in a request body before they reach route handlers.

import { Request, Response, NextFunction } from 'express';
import { parseISO, isValid } from 'date-fns';

type DateField = string;

export function validateDateFields(fields: DateField[]) {
  return (req: Request, res: Response, next: NextFunction): void => {
    for (const field of fields) {
      const value = req.body[field];

      if (value === undefined || value === null) continue; // optional field

      if (typeof value !== 'string' || !isValid(parseISO(value))) {
        res.status(400).json({
          error: `Field "${field}" must be a valid ISO 8601 date string. Received: ${JSON.stringify(value)}`,
        });
        return;
      }
    }

    next();
  };
}

Usage:

import express from 'express';
import { validateDateFields } from './middleware/validateDateFields';

const router = express.Router();

router.post(
  '/events',
  validateDateFields(['startsAt', 'endsAt']),
  async (req, res) => {
    const startsAt = new Date(req.body.startsAt); // already validated — safe to construct
    // ... save to DB
  }
);

Prisma: Store and Read UTC Timestamps​

Full example of writing a UTC timestamp and reading it back as an ISO 8601 string.

import { PrismaClient } from '@prisma/client';

const prisma = new PrismaClient();

interface CreateEventInput {
  title: string;
  startsAt: string; // ISO 8601 UTC string from the API
}

interface EventResponse {
  id: string;
  title: string;
  startsAt: string; // ISO 8601 UTC string returned to the API
  createdAt: string;
}

export async function createEvent(input: CreateEventInput): Promise<EventResponse> {
  const startsAtUtc = new Date(input.startsAt); // normalize to Date (UTC)

  const event = await prisma.event.create({
    data: {
      title: input.title,
      startsAt: startsAtUtc,
    },
  });

  return {
    id: event.id,
    title: event.title,
    startsAt: event.startsAt.toISOString(),  // always return ISO 8601
    createdAt: event.createdAt.toISOString(),
  };
}

The matching Prisma schema:

model Event {
  id        String   @id @default(cuid())
  title     String
  startsAt  DateTime @db.Timestamptz(3)
  createdAt DateTime @default(now()) @db.Timestamptz(3)
  updatedAt DateTime @updatedAt      @db.Timestamptz(3)
}

Edge Cases: Invalid Input Handling​

Handle the most common failure modes without throwing exceptions.

import { parseISO, isValid, isAfter, isBefore } from 'date-fns';

interface DateRangeValidationResult {
  valid: boolean;
  startDate: Date | null;
  endDate: Date | null;
  error: string | null;
}

export function validateDateRange(
  startIso: string,
  endIso: string
): DateRangeValidationResult {
  const start = parseISO(startIso);
  const end = parseISO(endIso);

  if (!isValid(start)) {
    return { valid: false, startDate: null, endDate: null, error: `Invalid start date: "${startIso}"` };
  }

  if (!isValid(end)) {
    return { valid: false, startDate: null, endDate: null, error: `Invalid end date: "${endIso}"` };
  }

  if (!isBefore(start, end)) {
    return {
      valid: false,
      startDate: null,
      endDate: null,
      error: `Start date must be before end date. Got: ${startIso} → ${endIso}`,
    };
  }

  return { valid: true, startDate: start, endDate: end, error: null };
}

Usage:

// ✅ Valid range
validateDateRange('2024-03-10T00:00:00.000Z', '2024-03-15T00:00:00.000Z');
// { valid: true, startDate: Date, endDate: Date, error: null }

// ❌ End before start
validateDateRange('2024-03-15T00:00:00.000Z', '2024-03-10T00:00:00.000Z');
// { valid: false, ..., error: "Start date must be before end date." }

// ❌ Malformed input
validateDateRange('not-a-date', '2024-03-10T00:00:00.000Z');
// { valid: false, ..., error: 'Invalid start date: "not-a-date"' }

Never accept timezone-naive strings from external clients. A string like "2024-03-10T14:30:00" (missing the Z) is ambiguous. Reject it or treat it as UTC explicitly — do not silently assume local time.
